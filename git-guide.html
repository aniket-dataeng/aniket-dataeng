1. Initializing a Repository

To start tracking your project with Git, navigate to the project directory and run:

git init

2. Connecting to a Remote Repository

Link your local project to a remote GitHub repository:

git remote add origin https://github.com/your-username/your-repo.git

3. Checking the Status

See which files are untracked or modified:

git status

4. Staging and Committing Changes

Stage files before committing:

git add .

Commit with a message:

git commit -m "Initial commit"

5. Pushing to GitHub

Push your code to GitHub:

git push origin main

6. Pulling Latest Changes

Always pull the latest changes before pushing to avoid conflicts:

git pull origin main --rebase

7. Handling Multiple Repositories

Each folder should be linked to only one repository. If you have multiple projects, keep them in separate directories, each with its own Git repository.
